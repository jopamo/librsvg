# librsvg API and ABI Reference

## Overview

librsvg is a SAX-based SVG rendering library built on GLib/GObject. It provides APIs for loading, parsing, and rendering SVG files to multiple backends (Cairo surfaces, GdkPixbuf). This document covers the public API (version 2.0) and ABI stability guarantees for developers creating or maintaining software that uses librsvg.

**API Version**: 2.0 (stable)
**Library Version**: 2.40.21 (example)
**SONAME**: `librsvg-2.so` (soversion: 2)

## Header Files

### Public Headers (Installed)

These headers are installed to `${includedir}/librsvg-2.0/librsvg/` and constitute the public API:

1. **`rsvg.h`** – Primary public API
   - Handle management
   - Dimension and position queries
   - GIO integration
   - GdkPixbuf rendering
   - DPI configuration

2. **`rsvg-cairo.h`** – Cairo rendering API
   - Requires Cairo context
   - Linked from `rsvg.h` automatically

3. **`rsvg-deprecated.h`** – Deprecated APIs
   - Included automatically via `rsvg.h`
   - Do not include directly

### Generated Headers (Installed)

4. **`librsvg-features.h`** – Version macros and feature detection
   - Generated from `src/librsvg-features.h.in`
   - Contains version constants and feature flags

5. **`librsvg-enum-types.h`** – GObject enumeration type definitions
   - Generated by `glib-mkenums` from `rsvg.h` and `rsvg-cairo.h`
   - Provides `GType` registration functions

### Internal Headers (Not Installed)

- `rsvg-private.h` – Internal types and functions (marked `G_GNUC_INTERNAL`)
- Module-specific headers (`rsvg-filter.h`, `rsvg-shapes.h`, etc.)
- **Do not use these in external code**

## Data Types and Structures

### Opaque Types

```c
typedef struct _RsvgHandle RsvgHandle;           // Main SVG handle
typedef struct RsvgHandlePrivate RsvgHandlePrivate;  // Private data (opaque)
typedef struct _RsvgHandleClass RsvgHandleClass;     // Class structure
```

### Public Structures

#### `RsvgDimensionData` – Dimensions of SVG or SVG fragment

```c
struct _RsvgDimensionData {
    int width;      // Width in pixels
    int height;     // Height in pixels
    gdouble em;     // Em unit (font-relative)
    gdouble ex;     // Ex unit (font-relative)
};
```

#### `RsvgPositionData` – Position of SVG fragment

```c
struct _RsvgPositionData {
    int x;  // X coordinate
    int y;  // Y coordinate
};
```

### Class and Instance Structures (ABI-Padded)

```c
struct _RsvgHandleClass {
    GObjectClass parent;
    gpointer _abi_padding[15];  // Reserved for future virtual functions
};

struct _RsvgHandle {
    GObject parent;
    RsvgHandlePrivate* priv;    // Private implementation data
    gpointer _abi_padding[15];  // Reserved for future instance data
};
```

**ABI Note**: The 15 `gpointer` padding slots in both structures allow adding virtual methods and instance data without breaking existing binaries.

## Enumerations

### `RsvgError` – Error domain enumeration

```c
typedef enum {
    RSVG_ERROR_FAILED   // General failure (only error code)
} RsvgError;
```

### `RsvgHandleFlags` – Bit flags for handle creation (since 2.40.3)

```c
typedef enum {
    RSVG_HANDLE_FLAGS_NONE = 0,
    RSVG_HANDLE_FLAG_UNLIMITED = 1 << 0,       // Disable security limits (trusted input only!)
    RSVG_HANDLE_FLAG_KEEP_IMAGE_DATA = 1 << 1  // Retain image data for Cairo PDF output
} RsvgHandleFlags;
```

**Security Warning**: `RSVG_HANDLE_FLAG_UNLIMITED` disables DoS protections. Use only for trusted SVG content.

## Error Handling

### Patterns

1. **Functions returning `gboolean`**: Return `TRUE` for success, `FALSE` for failure with `GError**` set.
2. **Functions returning pointers**: Return `NULL` on error with `GError**` set.
3. **Error domain**: `RSVG_ERROR` (obtained via `rsvg_error_quark()`).
4. **Single error code**: `RSVG_ERROR_FAILED` (general failure).

### Example

```c
GError *error = NULL;
RsvgHandle *handle = rsvg_handle_new_from_file("image.svg", &error);
if (!handle) {
    g_printerr("Failed to load SVG: %s\n", error->message);
    g_error_free(error);
    return;
}

/* Use handle... */
g_object_unref(handle);
```

## ABI Stability and Versioning

### ABI Padding Mechanism

Both `RsvgHandleClass` and `RsvgHandle` contain 15 `gpointer` padding slots:

```c
gpointer _abi_padding[15];  // In both class and instance structures
```

These slots are reserved for:
- Future virtual methods (class padding)
- Future instance data (instance padding)
- Allow ABI-compatible extensions without breaking existing binaries

### Version Scheme

- **API version**: 2.0 (stable interface)
- **Library version**: Semantic versioning (e.g., 2.40.21)
- **SONAME**: `librsvg-2.so` (soversion: 2)
- **pkg-config name**: `librsvg-2.0`

### Build Configuration (Meson)

From `meson.build`:

```meson
api_version = '2.0'
api_major = '2'

library('rsvg-' + api_major,
        soversion: api_major,           # SONAME: librsvg-2.so
        version: meson.project_version(),  # e.g., 2.40.21
        install: true)
```

### Header Installation

Headers installed to:


Include path in code:

## Security Considerations

### Built-in Protections

1. **Element reference limit**: 1024 per document (DoS protection)
2. **XSS protection**:
   - Ignores `<script>` tags
   - Blocks `javascript:` URLs
   - Ignores event handlers (`onclick`, `onload`, etc.)
3. **Memory limits**: Default parsing limits on document complexity

### Unlimited Mode

The `RSVG_HANDLE_FLAG_UNLIMITED` flag disables all security limits:

```c
RsvgHandle *handle = rsvg_handle_new_with_flags(RSVG_HANDLE_FLAG_UNLIMITED);
```

**Warning**: Use only for trusted SVG content. Malicious SVG files with this flag can cause:
- Excessive memory consumption
- CPU exhaustion via complex paths
- Other DoS attacks

### Testing

Security tests in `tests/vulnerability_check.c` validate:
- Various attack vectors
- XSS protection
- Resource exhaustion attempts
- Parser edge cases

## Deprecated API Reference

**File**: `rsvg-deprecated.h` (included automatically via `rsvg.h`)

### Initialization (Replaced by GLib)

```c
RSVG_DEPRECATED_FOR(g_type_init) void rsvg_init(void);
RSVG_DEPRECATED void rsvg_term(void);
```

**Modern alternative**: GLib initializes automatically.

### Memory Management (Replaced by GObject)

```c
RSVG_DEPRECATED_FOR(g_object_unref) void rsvg_handle_free(RsvgHandle* handle);
```

**Modern alternative**:
```c
g_object_unref(handle);  // Use GObject reference counting
```

### Size Callback (Replaced by Cairo Matrix)

```c
typedef void (*RsvgSizeFunc)(gint* width, gint* height, gpointer user_data);

                                                   RsvgSizeFunc size_func,

                                                   GDestroyNotify user_data_destroy);
```

**Modern alternative**: Use Cairo transformation matrix with `rsvg_handle_render_cairo()`.

## Compatibility Guidelines

### API Compatibility

- **Stable API**: Version 2.0 API is stable
- **New functions**: May be added in minor releases
- **Behavior changes**: Only in major releases with deprecation cycles
- **Deprecated functions**: Remain available but marked with `RSVG_DEPRECATED`

### ABI Compatibility

- **Guaranteed**: Within same soversion (2)
- **Breaking changes**: Require new soversion (e.g., librsvg-3.so)
- **Padding slots**: Ensure forward compatibility

### Version Checking

**Compile-time**:
```c
#include <librsvg/librsvg-features.h>

#if LIBRSVG_CHECK_VERSION(2, 40, 0)
/* Use features from 2.40.0+ */
#endif
```

**Run-time**:
```c
extern const guint librsvg_major_version;
extern const guint librsvg_minor_version;

if (librsvg_major_version > 2 ||
    (librsvg_major_version == 2 && librsvg_minor_version >= 40)) {
    /* Use 2.40.0+ features */
}
```

### Building Against librsvg

**Minimum example** (`example.c`):
```c
#include <librsvg/rsvg.h>
#include <stdio.h>

int main(int argc, char *argv[]) {
    GError *error = NULL;
    RsvgHandle *handle;
    RsvgDimensionData dimensions;

    if (argc < 2) {
        fprintf(stderr, "Usage: %s <svg-file>\n", argv[0]);
        return 1;
    }

    handle = rsvg_handle_new_from_file(argv[1], &error);
    if (!handle) {
        fprintf(stderr, "Error: %s\n", error->message);
        g_error_free(error);
        return 1;
    }

    rsvg_handle_get_dimensions(handle, &dimensions);
    printf("SVG dimensions: %d x %d pixels\n", dimensions.width, dimensions.height);

    g_object_unref(handle);
    return 0;
}
```

**Compilation**:
```bash
gcc -o example example.c `pkg-config --cflags --libs librsvg-2.0`
```

## Common Patterns

### Loading and Rendering to Cairo

```c
#include <librsvg/rsvg.h>
#include <cairo/cairo.h>

void render_svg_to_surface(const char *filename, cairo_surface_t *surface) {
    GError *error = NULL;
    RsvgHandle *handle;
    cairo_t *cr;

    handle = rsvg_handle_new_from_file(filename, &error);
    if (!handle) {
        g_printerr("Failed to load SVG: %s\n", error->message);
        g_error_free(error);
        return;
    }

    cr = cairo_create(surface);
    rsvg_handle_render_cairo(handle, cr);

    cairo_destroy(cr);
    g_object_unref(handle);
}
```

### Loading and Getting GdkPixbuf

```c
#include <librsvg/rsvg.h>

GdkPixbuf *load_svg_as_pixbuf(const char *filename, int width, int height) {
    GError *error = NULL;
    RsvgHandle *handle;
    GdkPixbuf *pixbuf;

    handle = rsvg_handle_new_from_file(filename, &error);
    if (!handle) {
        g_printerr("Failed to load SVG: %s\n", error->message);
        g_error_free(error);
        return NULL;
    }

    /* Set DPI for desired size */
    rsvg_handle_set_dpi_x_y(handle, width, height);

    pixbuf = rsvg_handle_get_pixbuf(handle);
    if (pixbuf)
        g_object_ref(pixbuf);  /* Keep pixbuf after handle destruction */

    g_object_unref(handle);
    return pixbuf;
}
```

### Using GIO Streams

```c
#include <librsvg/rsvg.h>
#include <gio/gio.h>

void load_from_stream(GInputStream *stream) {
    GError *error = NULL;
    RsvgHandle *handle;

    handle = rsvg_handle_new_from_stream_sync(stream,
                                              NULL,  /* base_file */
                                              RSVG_HANDLE_FLAGS_NONE,
                                              NULL,  /* cancellable */
                                              &error);
    if (!handle) {
        g_printerr("Stream loading failed: %s\n", error->message);
        g_error_free(error);
        return;
    }

    /* Use handle... */
    g_object_unref(handle);
}
```

## Troubleshooting

### Common Issues

1. **Missing includes**: Ensure `#include <librsvg/rsvg.h>` (not `rsvg/rsvg.h`)
2. **Linker errors**: Verify pkg-config returns correct `-lrsvg-2` flag
3. **Runtime loading failures**: Check SVG file validity and library dependencies
4. **Memory leaks**: Use `g_object_unref()` not `rsvg_handle_free()`

### Debugging

Enable debug output:
```bash
G_MESSAGES_DEBUG=librsvg ./program
```

### Testing Compatibility

Test against multiple versions:
```bash

pkg-config --modversion librsvg-2.0
ldd /usr/lib/libresvg-2.so
```
